## 🧠 PROMPT — IMPLEMENT CV FILE IMPORT + AUTO-FILL (NO SUPABASE)

---

### 🎯 Objective

Implement a **CV file import and auto-fill system** in my Next.js (App Router) application.

The system must:

* Allow users to upload a CV (PDF or DOCX)
* Extract the text from the file
* Process the content using AI
* Return structured resume data
* Automatically fill the resume form

---

# ❗ CRITICAL REQUIREMENTS

* Remove ALL dependency on Supabase
* Do NOT store files externally
* Do NOT use database
* Process everything via server (API routes)
* Must work on Vercel

---

# 🧱 IMPLEMENTATION ARCHITECTURE

---

## 1️⃣ FILE UPLOAD (CLIENT)

Create file input:

* Accept:

  * .pdf
  * .docx

On upload:

* Send file to API route `/api/parse-cv`
* Use `FormData`

---

## 2️⃣ API ROUTE (SERVER)

Create:

```
/app/api/parse-cv/route.ts
```

---

## 3️⃣ FILE PARSING

### PDF

Use:

```
pdf-parse
```

### DOCX

Use:

```
mammoth
```

---

### Extract raw text from file

Output:

```
plain text string
```

---

## 4️⃣ AI PROCESSING

Send extracted text to AI (OpenAI or compatible API).

Prompt must extract structured data.

---

### REQUIRED OUTPUT FORMAT (STRICT JSON)

```json
{
  "personalInfo": {
    "fullName": "",
    "jobTitle": "",
    "email": "",
    "phone": "",
    "location": "",
    "linkedin": "",
    "website": ""
  },
  "summary": "",
  "experience": [
    {
      "company": "",
      "position": "",
      "startDate": "",
      "endDate": "",
      "description": ""
    }
  ],
  "education": [
    {
      "school": "",
      "degree": "",
      "startDate": "",
      "endDate": ""
    }
  ],
  "skills": []
}
```

---

## 5️⃣ RETURN DATA

Return JSON response to frontend.

---

## 6️⃣ AUTO-FILL FORM

On client:

* Receive structured data
* Merge into resume state
* Update UI instantly

---
